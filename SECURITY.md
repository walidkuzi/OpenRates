# Security policy

## Reporting a vulnerability

Do not open a public issue for security vulnerabilities.

Report privately using GitHub's private vulnerability reporting feature on this repository.
Alternatively, email security@openrates.dev (placeholder — check the repository for the
current contact).

Include:

- A description of the vulnerability.
- Steps to reproduce.
- The impact and your assessment of severity.
- Any proposed fixes.

## Response timeline

| Severity | Acknowledgment | Fix target |
|---|---|---|
| Critical | 48 hours | 7 days |
| High | 48 hours | 14 days |
| Medium | 48 hours | 30 days |
| Low | 48 hours | Next release |

## Supported versions

Security updates are provided for the current main branch. Once stable releases are
tagged, updates cover the most recent major version.

## What not to report publicly

- Zero-day vulnerabilities before a fix is available.
- Active exploitation details.
- Exposed API keys or credentials.
- Non-public deployment information.

## Safe harbor

We will not pursue legal action against researchers who:

- Report in good faith and follow responsible disclosure.
- Avoid accessing, modifying, or exfiltrating data beyond what is needed to prove the issue.
- Do not publicly disclose before a fix is released.
